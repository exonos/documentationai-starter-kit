<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## New Features

- Added new endpoint `/v1/validations/idse` for streamlined IDSE worker registration.
- Introduced bulk validation support in API Mexico, processing up to 100 records per request.
- Enhanced dashboard with real-time usage analytics and API key management.

## Improvements

- Reduced average response time by 40% for high-volume validation requests.
- Added support for additional Mexican legal registries (SAT, RFC, CURP).

## Bug Fixes

- Fixed intermittent 500 errors on `/v1/validations/curp` endpoint.
- Resolved token expiration issues during long-running bulk operations.

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## New Features

- Launched IDSE Pro API with endpoints for worker onboarding and compliance checks.
- Webhook support for validation results, enabling asynchronous notifications.

## Improvements

- Updated rate limits to 5000 requests per minute for enterprise plans.
- Improved error messages with detailed Mexican registry-specific codes.

## Bug Fixes

- Corrected RFC validation logic for recent SAT format changes.
- Fixed pagination issues in list endpoints.

<Callout kind="alert">
  The legacy `/v0/validations` endpoint is deprecated. Migrate to `/v1/validations` before 2024-12-01.
</Callout>

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## Initial Release

- Core API Mexico validation endpoints for CURP, RFC, and 15+ legal registries.
- Authentication via API keys with Bearer token support.
- Dashboard for monitoring API usage and generating reports.

## Breaking Changes

- Switched from HMAC signatures to Bearer tokens for authentication.

</Update>
