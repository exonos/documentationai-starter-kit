## Overview

apimarket.mx provides RESTful APIs for validating clients against Mexican legal registries and managing IDSE registrations. All endpoints use HTTPS and follow standard HTTP methods.

Base URL: `https://api.apimarket.mx/v1`

Authentication requires an API key passed in the `Authorization` header as `Bearer YOUR_API_KEY`.

<Callout kind="info">
  Generate your API key from the [dashboard](https://dashboard.apimarket.mx). Store it securely and rotate regularly.
</Callout>

## Authentication

## Validate Client

Validate a client's identity using CURP or RFC.

**Endpoint:** `POST https://api.apimarket.mx/v1/validate`

<Request tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.apimarket.mx/v1/validate', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      curp: 'GAAA900101HDFMLL00',
      rfc: 'GAAA900101HDF'
    })
  });
  const data = await response.json();
  ```
  ```python
  import requests
  response = requests.post(
    'https://api.apimarket.mx/v1/validate',
    headers={
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    json={
      'curp': 'GAAA900101HDFMLL00',
      'rfc': 'GAAA900101HDF'
    }
  )
  data = response.json()
  ```
  ```bash
  curl -X POST https://api.apimarket.mx/v1/validate \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "curp": "GAAA900101HDFMLL00",
      "rfc": "GAAA900101HDF"
    }'
  ```
</Request>

### Request Parameters

<ParamField body="curp" param-type="string" required="false">
  Mexican CURP identifier (18 characters).
</ParamField>

<ParamField body="rfc" param-type="string" required="false">
  RFC identifier. At least one of `curp` or `rfc` required.
</ParamField>

<ParamField body="registries" param-type="array" required="false">
  Specific registries to check, e.g., `["ine", "sat", "imss"]`. Defaults to all.
</ParamField>

### Response Format

<Response tabs="200,400">
  ```json
  {
    "success": true,
    "data": {
      "curp": {
        "valid": true,
        "details": {
          "name": "Juan Pérez García",
          "birthdate": "1989-01-01",
          "registries": ["ine", "sat"]
        }
      }
    }
  }
  ```
  ```json
  {
    "success": false,
    "error": {
      "code": "INVALID_CURP",
      "message": "CURP format is invalid"
    }
  }
  ```
</Response>

<ResponseField name="success" field-type="boolean" required="true">
  Indicates if validation succeeded.
</ResponseField>

<ResponseField name="data" field-type="object">
  Validation results per identifier.
</ResponseField>

<ResponseField name="error.code" field-type="string">
  Error code for failures.
</ResponseField>

## Error Handling

All errors return a standard JSON format.

<Callout kind="alert">
  Rate limits: 1000 requests per minute. Exceeding triggers `429 Too Many Requests`.
</Callout>

Common codes:

| Code          | Description                  |
|---------------|------------------------------|
| `INVALID_PARAM` | Missing or invalid parameter |
| `UNAUTHORIZED` | Invalid API key             |
| `NOT_FOUND`   | Identifier not found        |

## IDSE Registration

**Endpoint:** `POST https://api.apimarket.mx/v1/idse/register`

<Tabs>
